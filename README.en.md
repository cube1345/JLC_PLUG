# Silk Generator

An extension for JLCEDA / EasyEDA Pro PCB editor that generates and manages silkscreen labels for pin headers and female headers from pad net names.

## Features

- Detects the selected header component and can also resolve the parent component from a selected pad.
- Extracts compact labels from pad net names and falls back to `P1`, `P2`, and similar pin numbers when no net is available.
- Supports custom label mappings with one rule per line, for example `GND=G` and `PA15=A15`.
- Automatically analyzes orientation, single-row or dual-row structure, row grouping, and shell bounds.
- Provides a compact tabbed panel for font, units, layer, font size, stroke width, placement side, rotation, offset, shell outline, and inverted text.
- Includes a preview on the layout tab while keeping the panel footprint small enough to reduce PCB canvas obstruction.
- Generates labels by letting you define a placement range on the PCB canvas, then distributing each label across that range instead of collapsing the whole group onto one point.
- Automatically shrinks labels slightly in dense areas based on your configured size to reduce overlap.
- Supports deleting a generated silk group for the selected header; older groups without records fall back to range-based deletion.
- Uses net parsing and geometry caching so repeated generation on the same header becomes faster.

## Generation Workflow

1. In the PCB editor, select one pin header, female header, or any pad that belongs to it.
2. Open the top menu `丝印一键生成 > 生成丝印...`.
3. Adjust settings in the `Quick` and `Layout` tabs. If needed, enter custom label mappings in the mapping box.
4. Click `Generate`.
5. Return to the PCB canvas and define a rectangular placement range.
6. The extension distributes the labels across that range and auto-shrinks dense labels when needed.

## Deletion Workflow

1. In the PCB editor, select a header that already has generated silk.
2. Open the same panel.
3. Click `Delete`.
4. If the silk group was generated by the current version, it is removed directly from the stored record.
5. If the group comes from an older version without stored records, the extension asks you to define a deletion range on the PCB canvas.

## Label Mapping Syntax

Enter one mapping rule per line in the mapping box on the `Layout` tab:

```text
GND=G
PA15=A15
USART1_TX=TX
```

Notes:

- The left side is the parsed source label.
- The right side is the final displayed silk label.
- `=`, `:`, `->`, and `=>` are all accepted as separators.
- Empty lines and lines starting with `#` or `//` are ignored.

## Configuration Options

- Font: choose the font used for silkscreen text.
- Units: switch numeric input and display between `mil` and `mm`.
- Layer: follow component side automatically, or force top or bottom silkscreen.
- Font size and stroke width: control overall text size and line thickness.
- Relative placement and rotation: define which side of the component receives the labels and whether rotation is automatic or fixed.
- Offset: control the distance between labels and the component body.
- Shell outline: add an outline around the generated label group.
- Invert: render text with an inverted visual style.
- Label mapping: replace parsed labels with your preferred abbreviations.

## Limits

- Works only inside the PCB editor.
- Processes one header at a time. Selecting multiple components triggers a warning.
- Requires readable pad and net data from the selected component.
- Custom mappings apply to parsed labels, not to the full original net path.

## Development

Requirements:

- Node.js `>= 20.17.0`
- JLCEDA / EasyEDA Pro extension runtime `^3.0.0`

Build locally:

```bash
npm install
npm run build
```

The build output is a `.eext` package under `build/dist/`, ready to import into JLCEDA / EasyEDA Pro.

## Project Layout

- `src/index.ts`: extension entry, menu registration, and panel launch.
- `iframe/header-silk.html`: panel markup.
- `iframe/js/header-silk.js`: component detection, net parsing, preview, mapping, deletion, and silkscreen generation logic.
- `iframe/css/header-silk.css`: panel styling.
- `build/packaged.ts`: packaging script that produces the `.eext` file.
- `locales/`: translation resources for extension metadata and prompts.
